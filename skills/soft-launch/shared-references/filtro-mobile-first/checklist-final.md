# Checklist Final

Verificação obrigatória antes de publicar qualquer peça visual.

Esse é o teste de qualidade do método Soft Business. Sem ele, não vai pro ar.

---

## Passo 1 - Tipografia

- [ ] Corpo de texto ≥ 18px no mobile
- [ ] H1 ≥ 32px no mobile
- [ ] H2 ≥ 26px no mobile
- [ ] Texto de apoio (FAQ, rodapé, data) ≥ 16px
- [ ] Máximo 2 famílias de fonte
- [ ] Máximo 3 pesos por família
- [ ] Hierarquia clara (H1 > H2 > H3 > corpo)
- [ ] Bold usado com economia (1-2x por parágrafo)
- [ ] Italic só pra citação ou ênfase semântica
- [ ] All caps apenas em CTAs e tags curtas

---

## Passo 2 - Contraste e cor

- [ ] Contraste passa WCAG AA (4.5:1 pra corpo, 3:1 pra título grande)
- [ ] Validado em ferramenta (WebAIM Contrast Checker)
- [ ] Sem cinza claro/médio sobre branco
- [ ] Sem cinza escuro sobre preto
- [ ] Acento de cor aparece em 5-15% do design
- [ ] Botões têm 3 estados visíveis (default, hover, active)
- [ ] Sem cor neon ou pastel infantil
- [ ] Sem gradientes complexos atrapalhando leitura

---

## Passo 3 - Espaçamento e layout

- [ ] Line-height do corpo 1.5-1.7
- [ ] Margens laterais ≥ 24px no mobile
- [ ] Bloco pra bloco ≥ 64px de espaço
- [ ] Largura de linha ≤ 65 caracteres no desktop
- [ ] Parágrafos ≤ 5 linhas sem respiro
- [ ] Espaço em branco ≥ 30% da página
- [ ] Alinhamento à esquerda (95% dos casos)
- [ ] 1 coluna no mobile
- [ ] Vertical rhythm consistente (múltiplos de 8px ou 16px)
- [ ] Sem pop-ups que cobrem tela inteira

---

## Passo 4 - CTAs

- [ ] Largura ≥ 80% da tela no mobile
- [ ] Altura ≥ 56px
- [ ] Padding interno generoso (16-24px vertical + 32-48px horizontal)
- [ ] Contraste cor/fundo passa WCAG
- [ ] 3 estados visíveis (default, hover, active)
- [ ] Espaço ao redor ≥ 32px
- [ ] Texto imperativo, claro, 2-5 palavras
- [ ] 3-5 CTAs na página, sendo 1 primário claro
- [ ] CTA primário visível acima da dobra
- [ ] Sem cringe ou frase de coach

---

## Passo 5 - Performance

- [ ] Página carrega em < 3s no 3G simulado
- [ ] Imagens otimizadas (WebP/AVIF + comprimidas)
- [ ] Imagens com lazy loading abaixo da dobra
- [ ] font-display: swap nas fontes
- [ ] JavaScript mínimo, com defer/async
- [ ] HTTPS configurado
- [ ] PageSpeed Insights ≥ 80 mobile
- [ ] Sem vídeo pesado auto-play

---

## Passo 6 - Teste no celular REAL

Não no DevTools. Celular real.

- [ ] Aberto no celular real (Android e iPhone se possível)
- [ ] Lê cada bloco sem zoom
- [ ] CTAs clicáveis com polegar sem precisão milimétrica
- [ ] Scroll suave do topo até o final
- [ ] Sem engasgos
- [ ] Sem elementos que pulam
- [ ] Sem pop-up irritante

---

## Passo 7 - Acessibilidade básica

- [ ] Imagens com alt-text
- [ ] Botões com texto descritivo (não só ícone)
- [ ] Hierarquia de heading correta (H1 → H2 → H3, sem pular)
- [ ] Links visivelmente diferenciados (cor ou underline)
- [ ] Foco visível em elementos clicáveis (pra navegação por teclado)
- [ ] Contraste validado em ferramenta

---

## Passo 8 - Anti-IA (cruzar com `shared-references/filtro-anti-ia/`)

- [ ] Zero em dashes "—" no texto
- [ ] Nenhuma frase-emoldura
- [ ] Nenhum verbo genérico de transformação
- [ ] Teste em voz alta foi feito
- [ ] Soa como pessoa real

---

## Passo 9 - Funcional

- [ ] Todos os links funcionam
- [ ] Formulário envia corretamente
- [ ] E-mail de confirmação chega
- [ ] CTA leva ao destino certo (checkout, WhatsApp, etc)
- [ ] Tracking (analytics, pixel) funcionando

---

## Passo 10 - Mensagem central

Esse é o teste de fechamento.

- [ ] A mensagem central da página é clara em 5 segundos?
- [ ] O cliente alvo (a persona-âncora) reconhece o próprio problema?
- [ ] O método/solução aparece com nome claro?
- [ ] A oferta é compreensível (o que é, quanto custa, como receber)?
- [ ] Existe motivo pra agir agora (sem urgência fake)?

---

## Se algum item falhar

**Não publica.**

Refaz, testa de novo, recheca.

Pode parecer chato. Mas é o que separa página de vendas que converte de página de vendas que parece amadora.

---

## Tempo médio do checklist

- Página simples (1 produto, 1 CTA): **15-30 minutos**
- Página média (3-5 seções, FAQ): **30-60 minutos**
- Página longa (carta de vendas completa): **1-2 horas**

Investimento que economiza retrabalho e converte muito mais.

---

## Documentar o que passou

Pra cada peça aprovada, documentar:
- Data
- URL
- Score do PageSpeed
- Notas de ajustes feitos

Histórico ajuda na próxima peça.

---

## Quando o cliente quer publicar antes

Se o cliente pressiona pra publicar sem terminar o checklist:

1. Explica que cada item afeta conversão
2. Pergunta qual prazo é negociável
3. Se nada negociável, publica mas marca o que ficou pendente
4. Ajusta nos primeiros dias após publicação

Mas o ideal é não publicar incompleto.
