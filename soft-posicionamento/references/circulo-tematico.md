# Círculo Temático

> **Quando consultar:** sempre que estiver decidindo sobre **o que produzir** (não como, isso é função das skills de peça). Atravessa conteúdo, oferta, perfil, multiplataforma. É o mapa que define **o universo do que você tem permissão de falar**.
>
> **O que entrega:** decomposição do método em mapa de temas com 3 anéis (núcleo, direto, indireto) e periferia explícita. Toda peça produzida orbita um ponto desse mapa. Tema que não cabe dentro do círculo não vira peça, independente de quão bom seja.

## As 2 perguntas que o círculo responde

Toda criação no Soft Business, peça, produto, perfil, oferta, existe pra responder duas perguntas:

> **1. Por que alguém deveria me seguir?**
> **2. Por que alguém deveria comprar de mim?**

Sem círculo empacado, as duas respostas viram vagas. *"Porque eu sou bom no que faço"* ou *"porque eu entrego conteúdo de valor"* são respostas de creator, qualquer um pode dar.

Com círculo empacado, a resposta vira específica:

- **Por que me seguir?** Porque dentro deste círculo eu sou denso, consistente e repetitivo. Quem tá nesse universo encontra aqui o que não encontra em outro lugar.
- **Por que comprar de mim?** Porque cada peça desse círculo aponta pra uma lacuna que **só fecha no meu produto**.

A pergunta também é o **filtro de manutenção**: toda vez que tu hesita se uma peça deve sair, volta às duas perguntas. Resposta clara = peça vai. Resposta vaga = reconstrói o círculo.

## O que é o círculo

Mapa do universo temático em volta do método, com 3 anéis e uma periferia.

```
                 ┌─────────────────────────────┐
                 │       PERIFERIA             │
                 │   (não entra nunca)         │
                 │                             │
                 │  ┌───────────────────────┐  │
                 │  │   ANEL INDIRETO       │  │
                 │  │  (vida do cliente     │  │
                 │  │   cruzando método)    │  │
                 │  │                       │  │
                 │  │  ┌─────────────────┐  │  │
                 │  │  │  ANEL DIRETO    │  │  │
                 │  │  │ (toca o método) │  │  │
                 │  │  │                 │  │  │
                 │  │  │  ┌───────────┐  │  │  │
                 │  │  │  │  NÚCLEO   │  │  │  │
                 │  │  │  │  MÉTODO   │  │  │  │
                 │  │  │  └───────────┘  │  │  │
                 │  │  └─────────────────┘  │  │
                 │  └───────────────────────┘  │
                 └─────────────────────────────┘
```

### Núcleo, o método decomposto

O método em si, fragmentado em **todas as suas peças constituintes**: cada módulo, cada aula, cada framework, cada bônus, cada ferramenta.

Tudo que está dentro do produto é matéria-prima. Mateus chama de **"engenharia reversa do produto"**: pegar todos os módulos, todas as aulas, e botar num mapa mental, *"a gente consegue enxergar absolutamente tudo que tem no produto e daí fica mais fácil fazer conteúdo"*.

**Regra de construção do núcleo:**

- Lista exaustiva, não resumida (até bônus e templates entram)
- Cada item é nomeado com a palavra exata usada no produto
- Sem agrupamento, quanto mais granular, mais matéria-prima vira

### Anel direto, temas que tocam o método

Temas que são **a versão pública** dos pontos do núcleo. Cada item do núcleo gera 3-5 temas diretos.

Exemplo aplicado a um módulo "Carta Minimalista":

| Núcleo (módulo) | Temas diretos |
|---|---|
| Carta Minimalista | "Por que sua proposta detalhada perde pra carta de 4 blocos" |
| | "O bloco da Carta que mais fecha venda" |
| | "Carta com preço dentro vs preço fora, qual converte mais" |
| | "O erro de quem escreve carta como portfólio" |
| | "Sua carta tá longa demais, sinais e correção" |

Cada tema direto:
- Toca **um** ponto específico do núcleo
- Abre uma lacuna que **só fecha no método**
- Pode virar carrossel, reel, story, vídeo longo, qualquer formato

### Anel indireto, vida do cliente cruzando método

Temas que **não tocam o método diretamente**, mas orbitam a vida real do cliente ideal **E** cruzam o método em algum ponto.

A regra do cruzamento é dura: se o tema é só sobre a vida do cliente sem cruzamento com o método, **é creator**. Se cruza o método, é Soft.

Exemplo aplicado:

| Tema indireto | Por que está no círculo |
|---|---|
| "Por que comecei a recusar reuniões nas sextas" | Cruza com método, tempo livre é resultado de sistema (Sócio IA IA Vertical) |
| "A decisão que mudou meu jeito de cobrar" | Cruza com método, posicionamento de preço (Cap 3) |
| "O dia em que parei de aceitar cliente novo por indicação" | Cruza com método, filtro Soft (conversão) |
| "Como eu organizo minha rotina sem reunião" | Cruza com método, sistema constante (Sócio IA) |

Tema indireto **sem** cruzamento:

| Tema | Por que NÃO entra |
|---|---|
| "Minha rotina de café da manhã" | Não cruza com método, vida pura |
| "Filme que assisti no fim de semana" | Não cruza, entretenimento |
| "Política, esporte, opinião sobre celebridade" | Não cruza, opinião dispersa |

### Periferia, o que não entra nunca

A periferia precisa ser **explicitada**. Não basta saber o que entra, precisa saber o que **propositalmente fica de fora**.

A periferia tem 3 zonas:

| Zona da periferia | O que é | Exemplo |
|---|---|---|
| **Tema relacionado mas fora** | Coisa que parece estar no círculo mas não tá | Marketing pra e-commerce (o método é só pra serviço/consultoria) |
| **Tema interessante mas dispersor** | Coisa que tu domina mas que dispersa o leitor | Filosofia, livros, hobbies não-cruzados |
| **Tema viral mas estranho** | Trend do momento que não cruza método | Polêmica do dia, meme do mês |

**Regra dura:** uma vez definida a periferia, ela é regra. Tema da periferia **nunca entra** em peça pública, mesmo que seja excelente, mesmo que tu tenha opinião forte, mesmo que esteja bombando.

## Como construir o círculo (exercício)

### Passo 1, Decompor o produto (engenharia reversa)

Lista exaustiva, em uma sessão de 2-3 horas:

1. Abre o produto principal
2. Lista todos os módulos
3. Dentro de cada módulo, lista todas as aulas/seções
4. Lista todos os bônus, templates, ferramentas
5. Lista todos os frameworks nomeados (mesmo os que não viraram aula)

Saída esperada: 30-80 itens granulares.

### Passo 2, Gerar temas diretos

Pra cada item do núcleo, escreve 3-5 temas diretos.

Pergunta de geração:

- *"Que dúvida o cliente tem sobre [item] antes de comprar o produto?"*
- *"Que crença errada o mercado tem sobre [item]?"*
- *"Que dor essa parte do método resolve, e como nomear essa dor?"*

Saída esperada: 100-300 temas diretos.

### Passo 3, Gerar temas indiretos

Pra cada tema direto, escreve 2-3 temas indiretos cruzando vida do cliente:

- *"Que situação cotidiana do cliente toca esse tema?"*
- *"Que decisão pessoal minha (ou de outro cliente) ilustra esse tema?"*
- *"Que erro humano comum aparece nessa parte do método?"*

Saída esperada: 200-600 temas indiretos.

### Passo 4, Definir a periferia

Lista explícita do que **não entra**:

- Temas que parecem estar no círculo mas não estão
- Tópicos pessoais que dispersam (sem cruzamento)
- Trends e polêmicas externas
- Áreas adjacentes do mercado que tu não atende

Saída esperada: 20-50 itens proibidos.

### Passo 5, Documentar como super-documento

Junta tudo num documento único (Notion, Docs, Markdown). Esse é o **mapa de jogo**. Toda peça consulta esse documento antes de existir.

## Como o círculo guia toda criação

### Antes de produzir qualquer peça

Pergunta de filtro:

> *Essa peça toca qual ponto do círculo?*

| Resposta | Decisão |
|---|---|
| "Toca o item X do núcleo via tema direto Y" | ✅ produz |
| "Toca tema indireto Z que cruza módulo W" | ✅ produz |
| "É um tema que eu acho interessante mas não consigo localizar no círculo" | ❌ não produz, ou reconstrói o círculo se for omissão real |
| "Tá na periferia explícita" | ❌ não produz nunca |

### Reembalagem

Se um ponto do círculo já gerou peça que performou bem, não morre, **reembala**. Mesmo ponto, ângulo diferente, formato diferente, capa diferente. Soft v9 cap 2.7, *repetição insuportável é trabalho*. O círculo permite essa repetição **sem virar repetitivo** porque cada reembalagem entra por outro ângulo do mesmo ponto.

### Calendário

Distribui produção em pontos distintos do círculo. Em uma semana de 8 peças:

- Não repetir o mesmo ponto do núcleo mais de 2x
- Misturar anel direto (densidade) e anel indireto (humano/vida)
- Toda peça fechada com âncora explícita ou implícita ao núcleo

## Relação com outros princípios Soft

### Substitui parcialmente "aponta pro método" (cap 2.10)

| Cap 2.10 sozinho | Com círculo |
|---|---|
| "Toda peça aponta pro método ou faz seeding" | "Toda peça nasce dentro do círculo, apontamento já está embutido na escolha do tema" |
| Filtro **pós-produção** (olha a peça pronta e checa) | Filtro **pré-produção** (não produz tema fora do círculo) |
| Risco de virar regra mecânica no fim da peça | Costura natural porque o tema já é do território |

O cap 2.10 continua valendo como **checklist de fechamento**. Mas o círculo evita o problema antes, peça boa nasce de tema certo, não de costura forçada no final.

### Complementa "documentação > produção" (cap 2.5)

| Cap 2.5 sozinho | Com círculo |
|---|---|
| "Vive a vida e documenta" | "Vive a vida e documenta **dentro do círculo**" |
| Risco de documentar tudo virando creator | Documentação filtrada, só sai o que cruza método |

### Complementa Tipologia Transversal (cap 2.9)

A tipologia (Primário / Secundário / Vulnerabilidade Estratégica) define **o tom da peça**. O círculo define **o tema da peça**. Os dois eixos se cruzam:

| Eixo | O que define |
|---|---|
| **Círculo** | *Sobre o quê* a peça fala |
| **Tipologia** | *Em que tom* a peça fala |
| **Camada C1/C2/C3** | *Pra que função no funil* a peça serve |

Toda peça responde os 3 eixos antes de existir.

### Conecta com Camadas de Consciência atração

Cada camada usa pontos diferentes do círculo:

| Camada | Pontos preferenciais |
|---|---|
| **C1, Alcance** | Anel indireto (vida + cruzamento), tema com capa universal |
| **C2, Convicção** | Anel direto (toca método), fragmento do produto |
| **C3, Prova viva** | Núcleo aplicado em case real (cliente vivendo o método) |

Sem círculo, as camadas viram genéricas. Com círculo, cada camada vira **distribuição estratégica do mesmo universo**.

## Aplicação ao Perfil Enxuto (Cap 4)

O perfil é a **destilação do círculo em 3 elementos**:

| Elemento | Função no círculo |
|---|---|
| **Nome (SEO)** | Palavra-chave do nicho, primeira camada do círculo declarada |
| **Bio (4 linhas)** | Destila as 2 perguntas: pra quem é, qual problema, qual diferencial, qual "sem" |
| **Destaques** | Mapa visual do círculo: PROBLEMA (anel indireto + direto) → MÉTODO (núcleo) → CLIENTES (núcleo aplicado) |

A frase final da bio Soft é a versão pública do círculo: *"ajudo X a alcançar Y sem Z"*, onde X é o cliente do círculo, Y é o resultado do núcleo, Z é a periferia (o que NÃO se faz).

## Quando o círculo precisa ser refeito

Sinal de que o círculo precisa de revisão:

- **Tu tá hesitando muito** se uma peça deve sair → círculo está vago
- **Tudo que tu posta parece igual** → círculo está estreito demais (só núcleo, sem indireto)
- **Tu posta mas ninguém sabe o que tu vende** → círculo não está conectando ao produto
- **Tu posta sobre coisas demais** → círculo não foi declarado, ou periferia não existe
- **Cliente novo chega e pergunta "mas você atende [coisa fora do círculo]?"** → comunicação extra-círculo vazou

A revisão acontece a cada 3-6 meses ou quando o produto muda (módulo novo, oferta nova, reposicionamento).

## Exemplo aplicado: o círculo do Soft Business (auto-referência)

Pra ilustrar como funciona, mapa parcial do próprio círculo do Soft:

**Núcleo (parcial):**
- Posicionamento Incomum
- Carta Minimalista
- Carrossel 3C
- Stories CARO
- IA Vertical
- 7 Fases Soft (conversão)

**Anel direto (exemplos):**
- "Por que reunião gratuita treina cliente a não pagar"
- "O bloco da Carta que mais fecha venda"
- "Capa de carrossel que filtra cliente errado"
- "Caixinha respondida vence Caixinha aberta"
- "Como IA Vertical posta no teu lugar sem virar genérico"
- "Por que sua call de fechamento começa antes da call"

**Anel indireto (exemplos):**
- "Decidi parar de aceitar reunião nas sextas, e o que mudou no faturamento"
- "Como eu trabalho 5h por dia e fecho mais que quem trabalha 12"
- "O dia em que recusei cliente de R$50k e entendi meu posicionamento"
- "O que fiz quando minha agenda virou consultório gratuito"

**Periferia:**
- Marketing pra e-commerce
- Tráfego pago como pilar (é só ferramenta, não pilar)
- Lançamento como estratégia (só ferramenta tática)
- Política, religião, opinião pessoal não-cruzada
- Outros nichos de marketing (B2B SaaS, agência tradicional, etc.)

## Princípio de fechamento

Sem círculo, criação vira loteria. Com círculo, criação vira **disciplina aplicada a território claro**.

> *Quem fala de tudo é creator. Quem fala do círculo é referência.*

A diferença entre ser **mais um** e ser **a primeira pessoa que vem à cabeça quando alguém pensa no problema** é a disciplina de não sair do círculo, mesmo quando todo o feed tá puxando pra fora.
